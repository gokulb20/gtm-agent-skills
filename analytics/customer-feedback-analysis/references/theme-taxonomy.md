# Theme Taxonomy — Customer Feedback Analysis

## 8 Theme Classes

### 1. Pain-job-anchor
- **Signal:** Customer describes the underlying job they hired the product for
- **JTBD lens:** The core Job-to-be-Done
- **Routing:** → icp-definition (P-T-O refinement) + positioning-strategy

### 2. Forces-of-progress (push / pull / habit / anxiety)
- **Signal:** Why they switched / what pulled them / what held them / what worried them
- **Moesta 4-forces:** Sale happens when F1(push) + F2(pull) > F3(habit) + F4(anxiety)
- **Routing:** → positioning-strategy (counter-anxiety messaging) + icp-refinement-loop

### 3. Positive outcome / value realized
- **Signal:** "Saved X hours" / "ramp time dropped" / "team adopted in week 1"
- **Routing:** → kpi-reporting (value-realization KPIs) + marketing (case-study source)

### 4. Feature request
- **Signal:** "Wish you had X" / "we worked around X"
- **Routing:** → product backlog (Linear/Jira) + frequency aggregation

### 5. Bug / friction
- **Signal:** "Broke when X" / "couldn't figure out Y"
- **Routing:** → product backlog priority-sorted

### 6. Pricing reaction
- **Signal:** "Worth it" / "expensive at X" / specific dollar friction
- **Routing:** → revenue-forecasting ACV reality-check

### 7. Competitor mention
- **Signal:** Named competitor referenced (positive or negative)
- **Routing:** → competitive-intelligence (function-1)

### 8. Champion language (won-deal interviews)
- **Signal:** What the champion actually said to internal stakeholders to win the deal
- **Routing:** → cold-email-sequence framework defaults (use champion language in copy)

## Min-Corpus-Size Floor

5 feedback items per source (won interviews / churn / reviews) for aggregation.
Below → surface individual items only; no cross-corpus claims.
