# Routing Map — Customer Feedback Analysis

## Theme → Downstream Skill

| Theme class | Route to | What gets sent |
|---|---|---|
| Pain-job-anchor | icp-definition + positioning-strategy | Verified pain language + P-T-O anchor |
| Forces-of-progress | positioning-strategy + icp-refinement-loop | Push/pull/habit/anxiety patterns |
| Positive outcome | kpi-reporting + marketing | Value metrics + case-study-ready quotes |
| Feature request | product backlog (Linear/Jira) | Frequency + severity + verbatim |
| Bug/friction | product backlog | Priority-sorted + verbatim |
| Pricing reaction | revenue-forecasting | ACV reality-check signal |
| Competitor mention | competitive-intelligence | Named competitor + context + sentiment |
| Champion language | cold-email-sequence copy bank | Verbatim champion quotes for outbound copy |

## ICP-Implication Check

When a theme appears in ≥30% of won-deal interviews within a segment:
- Flag as `icp-implication`
- Route to `icp-refinement-loop` for next quarterly run
- Include: theme + segment + frequency + sample verbatim

## Product Backlog Routing

- Auto-route to Linear if `LINEAR_API_KEY` set
- Else: surface for manual triage with frequency + severity + verbatim context
- Include: "12 customers in segment Y, here's the verbatim" — not just "user wants X"
