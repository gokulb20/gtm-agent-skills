# Category Frame Choices

## Options (after Play Bigger / Dunford)

| Frame type | When to use | Risk |
|---|---|---|
| Existing established category | Buyer recognizes the label; you're a better option | Easier to position; harder to differentiate from incumbents |
| Subcategory of existing | Buyer recognizes parent; you're specialized | Right balance for most products; "X for Y" pattern |
| New category | Buyer doesn't yet recognize; you're creating the frame | High risk, high reward; expensive; only justified when alternatives genuinely don't fit |
| Cross-category | Buyer compares against multiple unrelated categories | Confusing; usually means positioning is unfocused |

## Decision Rule

**Default to subcategory.** New-category claims need explicit evidence and budget (10–18 months of category-creation work + paid education budget).

## Validation Tests

- **5-second test:** Does the buyer recognize the category in 5 seconds?
- **Search test:** Does anyone Google this term?
- **Competition test:** Are there G2 categories or analyst reports for it?

## Worked Example — WorkflowDoc (fictional)

Frame type: **Subcategory of existing** ("AI-native support runbook authoring" within "knowledge management for support")
- 5-second test: Pass — buyers recognize "support runbooks"
- Search test: Pass — "runbook software for support" has volume
- G2/analyst recognition: Partial — G2 has KM parent; sub-category is emerging
