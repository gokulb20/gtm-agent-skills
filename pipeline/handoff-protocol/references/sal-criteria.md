# SAL Acceptance Criteria — Handoff Protocol

## SiriusDecisions / Forrester SAL Norms

SAL (Sales Accepted Lead) is rejected only for procedural / clerical / definitional reasons. Otherwise accepted with 24–72h follow-up SLA.

## 4 Definitional Acceptance Criteria

| # | Criterion | Check |
|---|---|---|
| 1 | ICP fit confirmed | Score meets Tier-1 or Tier-2 threshold per icp-definition rubric |
| 2 | Trigger present and within half-life | Trigger event detected and not expired (e.g., VP hire within 90d) |
| 3 | Decision-maker / Champion identified | Role mapped per discovery-call-prep; at least champion identified |
| 4 | No hard disqualifiers | Anti-ICP firmographic, DNC, role-based email — any present = disqualify |

## Pre-Delivery Gate

All 4 criteria MUST pass before the handoff package is delivered. If any fail:
- Route back to SDR with the specific gap
- SDR fixes the gap (extract from history, schedule discovery, re-enrich)
- SDR re-submits the handoff

## Rejection Reasons (when SAL criteria pass but rep rejects)

Common procedural rejection reasons:
- "Missing decision_process slot" — clerical, SDR can extract
- "Wrong segment for me" — routing error, reassign to correct AE
- "Briefing stale" — re-prep needed

## Acceptance SLA

- Rep has 24h to respond (accept or reject)
- No response within 24h → escalate (Slack ping, manager flag)
- Never auto-accept on silence
