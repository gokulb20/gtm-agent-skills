# DNS & Authentication Frameworks

## SPF (Sender Policy Framework) — RFC 7208

Publishes which IPs are authorized to send on behalf of the domain.

```
v=spf1 include:smtp.smartlead.ai -all
```

- Use `-all` (hard fail) for outbound domains
- Watch the 10-lookup limit — each `include` counts
- For multi-platform: `v=spf1 include:smtp.smartlead.ai include:_spf.google.com -all`

## DKIM (DomainKeys Identified Mail) — RFC 6376

Cryptographic signature proving the message wasn't altered in transit.

- **Minimum 2048-bit key** — Yahoo treats 1024-bit as deprecated; Google deprioritizes
- Selector format: `<selector>._domainkey.<domain>` (e.g. `smartlead._domainkey.workflowdoc-mail.com`)
- Key rotation: annually is fine; quarterly is overkill outside enterprise
- Generated by sending platform; user pastes CNAME/TXT record into DNS

## DMARC (Domain-based Message Authentication) — RFC 7489

Tells receivers what to do with non-authenticated mail and aggregates reports.

### Ramp schedule

| Phase | Policy | When |
|---|---|---|
| Day 0 | `p=none` | Start here — collect reports without blocking |
| Day+30 | `p=quarantine` | If reports clean for 30 days |
| High-stakes | `p=reject` | Only after quarantine is stable |

### Record format

```
v=DMARC1; p=none; rua=mailto:dmarc-reports@brand.com; ruf=mailto:dmarc-reports@brand.com; pct=100; aspf=r; adkim=r
```

- `rua` — aggregate report destination (daily)
- `ruf` — forensic report destination (per-message)
- `aspf=r; adkim=r` — strict alignment (recommended)

## MX Records

Route incoming reply traffic. Common patterns:

- Smartlead: `10 mx1.smartlead.ai`, `20 mx2.smartlead.ai`
- Google Workspace: `1 ASPMX.L.GOOGLE.COM`, `5 ALT1.ASPMX.L.GOOGLE.COM`
- M365: `0 <tenant>.mail.protection.outlook.com`

## rDNS / PTR

Reverse DNS on sending IPs. Sending platform usually owns this; verify.

## BIMI (Brand Indicators for Message Identification)

- Only after DMARC is `p=quarantine` or stricter
- Requires VMC (Verified Mark Certificate) — legal trademark registration
- Low priority; nice-to-have for brand recognition in inbox

## Google + Microsoft Feb 2024 Bulk Sender Requirements

Any sender to >5,000 messages/day to Gmail/Yahoo MUST:

1. Authenticate (SPF + DKIM + DMARC)
2. Maintain spam complaint rate **<0.3%** (warning at 0.1%)
3. Implement one-click List-Unsubscribe (RFC 8058)
4. Avoid spoofing Gmail in From:
5. From-address alignment: SPF and DKIM domains align with From: domain

Microsoft mirrors this for Outlook/Hotmail (rolling out 2024–2025). Non-compliance bounces with `550; 5.7.515 Access denied, sending domain does not meet the required authentication level.`
